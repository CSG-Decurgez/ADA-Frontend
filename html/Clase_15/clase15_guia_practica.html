Introducción

En esta guía veremos ejemplos prácticos de buenas y malas prácticas al nombrar y estructurar clases en CSS. Cada bloque incluye un ejemplo ❌ incorrecto y uno ✔️ correcto.

Usar nombres descriptivos

❌ .c1
✔️ .boton-primario
/* ❌ Evitar */
.c1 { background: blue; }

/* ✔️ Correcto */
.boton-primario { background: blue; }

Evitar abreviaciones

❌ .cnt
✔️ .contenedor
/* ❌ Evitar */
.cnt { padding: 20px; }

/* ✔️ Correcto */
.contenedor { padding: 20px; }

Evitar nombres demasiado largos

❌ .boton-azul-que-se-usa-en-el-header-principal
✔️ .btn-header
/* ❌ Evitar */
.boton-azul-que-se-usa-en-el-header-principal { ... }

/* ✔️ Correcto */
.btn-header { ... }

Usar kebab-case

❌ .botonPrimario
✔️ .boton-primario
/* ❌ Evitar */
.botonPrimario { ... }

/* ✔️ Correcto */
.boton-primario { ... }

Evitar selectores de etiqueta

❌ h1 { color: red; }
✔️ .titulo-principal { color: red; }
/* ❌ Evitar */
h1 { color: red; }

/* ✔️ Correcto */
.titulo-principal { color: red; }

Evitar IDs para estilos

❌ #header { background: black; }
✔️ .header { background: black; }
/* ❌ Evitar */
#header { background: black; }

/* ✔️ Correcto */
.header { background: black; }

Evitar selectores complejos

❌ .card ul li a span { color: blue; }
✔️ .card-link { color: blue; }
/* ❌ Evitar */
.card ul li a span { color: blue; }

/* ✔️ Correcto */
.card-link { color: blue; }

Usar prefijos para estados (is-, has-)

✔️ .is-activo
.is-activo { opacity: 1; }

Clases padre-hijo

✔️ .card-title
✔️ .card-body
.card-title { font-size: 20px; }
.card-body { padding: 12px; }

Ser consistentes

Si usás kebab-case, usalo en todo el proyecto.

/* ✔️ Correcto */
.btn-primario { ... }
.card-body { ... }
.titulo-principal { ... }

Reutilizar estilos

✔️ .text-center
.text-center { text-align: center; }

Usar comentarios y agrupar clases

/* --- Botones --- */
.btn { padding: 10px; }
.btn-primario { background: blue; }

/* --- Tarjetas --- */
.card { border: 1px solid #ddd; }
.card-body { padding: 12px; }
